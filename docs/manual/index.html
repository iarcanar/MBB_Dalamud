MBB Dalamud Project

คู่มืออ้างอิงสำหรับการพัฒนาต่อ — architecture, subsystems, ข้อตกลงที่ผ่านการพิสูจน์มาแล้ว และกฎที่ห้ามฝ่าฝืน. AI-readable + human-readable.

Project Goal

เปลี่ยน MBB Dalamud Bridge เป็นแพ็คเกจที่แจกจ่ายได้ผ่าน Dalamud Custom Plugin Repository — ผู้ใช้เพิ่ม URL ใน Dalamud settings และติดตั้งด้วย 1 คลิก

🧹 Code Cleanup

ลบ hardcoded paths ทั้งหมด, OCR-era code ถูก purge แล้ว, theme v2 + PyQt6 migration เสร็จสมบูรณ์

📦 Custom Repository

สร้าง pluginmaster.json ให้ Dalamud auto-discover plugin

⚡ PyInstaller

บรรจุ Python app เป็น standalone .exe พร้อม updater

🚀 One-Click Install

ติดตั้งผ่าน Dalamud plugin installer โดยไม่ต้อง setup เอง

Project Structure

C:\MBB_Dalamud/ ├── python-app/ # Python translation app │ ├── MBB.py # Entry point │ ├── translated_ui.py # TUI (Tkinter, dialog/choice) │ ├── translator_gemini.py # Gemini API + prompt v2 │ ├── npc.json # Character database (218 main / 65 npc / 135 lore) │ ├── pyqt_ui/ # PyQt6 panels + overlays │ ├── fonts/ # Anuphan, FC Minimal, Caveat, Pacifico, Google Sans │ └── assets/ # 60+ icons (white-themed, auto-invert on light) ├── DalamudMBBBridge/ # C# Dalamud plugin ├── docs/ # Landing page (index.html) + screenshots │ └── manual/ # ← You are here ├── scripts/ # build_npc_release.py + automation └── updater/ # Standalone updater (Tkinter)

Data Flow

FFXIV in-game text → Dalamud plugin → named pipe → Python translation pipeline → on-screen rendering. ทั้งหมดทำงานแบบ realtime.

FFXIV → Dalamud (C#) → Named Pipe → Python handler → translator + renderer

Build & Distribution

PyQt6 PINNED at 6.9.0

ห้าม upgrade PyQt6 เกิน 6.9.0 Qt 6.10+ มี frameless+translucent rendering regression — ทำให้ DissolveOverlay + Polaroid + main window กลายเป็นพื้นทึบสีดำ. ดูโปรโตคอลเต็มที่ BUILD_PROTOCOL.md

PyInstaller

📄 Specs

  • python-app/mbb.spec — main app
  • python-app/updater/updater.spec — updater

📁 Output

  • Build dir: dist_test/ (ไม่ใช่ python-app/dist/)
  • System Python 3.11

Post-build copy

หลัง COLLECT ใน mbb.spec จะ copy npc.json + npc_images/ ออกจาก _internal/:

MBB/ ├── MBB.exe ├── npc.json ← user-editable, visible ├── npc_images/ ← user data, visible └── _internal/ ├── npc.json (fallback) ├── npc_images/ (fallback) └── ... (Python runtime)

Frozen-Mode npc.json Rule (v1.8.7)

Any code touching npc.json MUST use get_npc_file_path()

resource_path() resolves to sys._MEIPASS/npc.json (immutable bundled snapshot) — fine for static assets (fonts, icons) แต่ ผิดสำหรับ npc.json ที่ user เขียนผ่าน NPC Manager. Resolver ต้องชี้ที่ exe-level file.

Verify: grep -rn 'resource_path.*npc' python-app/ ต้อง return zero matches
